# stack.qq 사용법

## 글 올리기

### 1. 파일 만들기

`posts/` 폴더 안에 `.md` 파일을 새로 만든다.  
파일 이름이 URL이 된다. 예: `my-review.md` → `~/posts/my-review`

띄어쓰기 없이, 영문 소문자와 하이픈(`-`)만 사용하는 것을 권장한다.

---

### 2. 상단 정보(frontmatter) 작성

파일 맨 위에 아래 형식으로 작성한다.

```yaml
---
title: "제목"
subtitle: "부제 (선택)"
category: "음악"
tags: ["태그1", "태그2"]
date: "2025-06-01"
thumbnail: "/thumbnails/파일명.jpg"
---
```

| 필드 | 설명 | 필수 여부 |
|------|------|-----------|
| `title` | 글 제목 | 필수 |
| `subtitle` | 부제목, 소개 문구 | 선택 |
| `category` | `음악` / `도서` / `영화` / `드라마` 중 하나 | 필수 |
| `tags` | 태그 배열 | 필수 (빈 배열 `[]` 가능) |
| `date` | 발행일 `YYYY-MM-DD` 형식 | 필수 |
| `thumbnail` | 썸네일 이미지 경로 | 선택 (없으면 카테고리 플레이스홀더 표시) |

---

### 3. 썸네일 이미지 추가

썸네일을 사용하려면 이미지 파일을 `public/thumbnails/` 폴더에 넣고, frontmatter에 경로를 입력한다.

```
public/
  thumbnails/
    my-review.jpg   ← 이미지를 여기에 넣는다
```

```yaml
thumbnail: "/thumbnails/my-review.jpg"
```

---

### 4. 본문 작성

frontmatter 아래부터 마크다운으로 본문을 작성한다.

```markdown
---
(frontmatter)
---

본문 내용을 여기에 작성한다.
```

---

## 마크다운 문법

| 문법 | 결과 |
|------|------|
| `**굵게**` | **굵게** |
| `*기울임*` | *기울임* |
| `## 소제목` | h2 소제목 (하단 구분선 포함) |
| `### 소소제목` | h3 소제목 |
| `> 인용문` | 왼쪽에 형광 노랑 선이 붙은 인용 블록 |
| `[링크텍스트](url)` | 하이퍼링크 |
| `` `코드` `` | 인라인 코드 |

---

## 글 수정하기

`posts/` 폴더에서 해당 `.md` 파일을 열어 직접 편집한다.  
frontmatter의 `date`는 자동으로 바뀌지 않으므로 필요 시 수동으로 갱신한다.

---

## 글 삭제하기

`posts/` 폴더에서 해당 `.md` 파일을 삭제한다.  
썸네일 이미지를 사용했다면 `public/thumbnails/`에서도 함께 지운다.

---

## 로컬에서 확인하기

```
시작 3000.bat
```

루트의 `시작 3000.bat`을 더블클릭하면 브라우저가 자동으로 열린다.

---

## 작성 예시

```markdown
---
title: "결말 이후에도 계속되는 영화"
subtitle: "고레에다 히로카즈 『어느 가족』 리뷰"
category: "영화"
tags: ["고레에다", "일본영화", "가족"]
date: "2025-06-09"
thumbnail: "/thumbnails/shoplifters.jpg"
---

고레에다 히로카즈의 영화는 언제나 **가족**이라는 단어를 다시 묻는다.

## 혈연 너머의 관계

> 이 영화에서 가족은 선택이다.

법적으로, 혹은 혈연으로 묶인 관계가 아니라 함께 살기로 *선택한* 사람들의 이야기.
```
